# Contributing to C# Code Samples

Thank you for your interest in contributing to this C# code samples repository!

## How to Contribute

This repository is designed to showcase clean code and best coding practices in C#. Contributions should align with these goals.

### What We're Looking For

- **Clean, readable code** that follows C# best practices
- **Well-documented examples** with clear explanations
- **SOLID principles** demonstrations
- **Design patterns** implementations
- **Real-world applicable examples** that solve common problems

### Contribution Guidelines

1. **Code Quality**
   - Follow C# naming conventions (PascalCase for classes/methods, camelCase for local variables)
   - Include XML documentation comments for public APIs
   - Keep methods small and focused (Single Responsibility Principle)
   - Write self-documenting code with meaningful names

2. **Documentation**
   - Each example should have a README.md explaining the concept
   - Include inline comments only when necessary to explain "why", not "what"
   - Provide usage examples

3. **Code Organization**
   - Place examples in appropriate directories
   - Each example should be self-contained
   - Include relevant using statements

4. **Testing**
   - Include unit tests where applicable
   - Demonstrate testable code design

### Getting Started

1. Fork the repository
2. Create a new branch for your contribution
3. Add your code samples following the structure
4. Submit a pull request with a clear description

### Code of Conduct

- Be respectful and constructive
- Focus on teaching and learning
- Welcome feedback and be open to improvements

## Questions?

Feel free to open an issue for any questions or suggestions!
