# C# Code Samples Repository

Welcome to the C# Code Samples repository! This is a collection of well-crafted C# code examples demonstrating clean code principles, design patterns, and best practices.

## üìö Repository Structure

### SOLID Principles
Demonstrations of the five SOLID principles for object-oriented design:
- **[Single Responsibility Principle (SRP)](./SOLIDPrinciples/SingleResponsibility/)** - A class should have only one reason to change
- **[Open/Closed Principle (OCP)](./SOLIDPrinciples/OpenClosed/)** - Open for extension, closed for modification
- **[Liskov Substitution Principle (LSP)](./SOLIDPrinciples/LiskovSubstitution/)** - Derived classes must be substitutable for their base classes
- **[Interface Segregation Principle (ISP)](./SOLIDPrinciples/InterfaceSegregation/)** - Clients should not depend on interfaces they don't use
- **[Dependency Inversion Principle (DIP)](./SOLIDPrinciples/DependencyInversion/)** - Depend on abstractions, not concretions

### Design Patterns
Classic design pattern implementations in C#:
- **[Creational Patterns](./DesignPatterns/Creational/)** - Object creation mechanisms
  - Factory Pattern
  - Singleton Pattern
  - Builder Pattern
- **[Structural Patterns](./DesignPatterns/Structural/)** - Object composition
  - Adapter Pattern
  - Decorator Pattern
- **[Behavioral Patterns](./DesignPatterns/Behavioral/)** - Object collaboration and responsibilities
  - Strategy Pattern
  - Observer Pattern

### Clean Code Examples
Practical examples of clean code practices:
- **[Naming Conventions](./CleanCode/Naming/)** - Meaningful and consistent naming
- **[Error Handling](./CleanCode/ErrorHandling/)** - Robust exception handling
